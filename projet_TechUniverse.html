PROJET : Optimisation de l'infrastructure DMZ - TechUniverse

Réalisé : Mars - Avril 2025
Projet BTS SIO - SISR

╔════════════════════╗
║ CONTEXTE DU PROJET ║
╚════════════════════╝
TechUniverse est une entreprise spécialisée dans la vente de produits technologiques.
Face à une forte hausse du trafic Web, elle souhaite moderniser son infrastructure DMZ afin de garantir la haute disponibilité de ses services et permettre le télétravail sécurisé.

╔═════════════════════════════════════════╗
║ MISSION 1 – MAQUETTE INFRASTRUCTURE DMZ ║
╚═════════════════════════════════════════╝
✔ Mise en place d’une maquette réseau avec :
- 2 serveurs Web (Apache)
- 1 pare-feu UTM Stormshield (WAN, LAN, DMZ)
✔ Règles NAT configurées pour rediriger le port 80 vers les serveurs web
✔ Validation de l’accessibilité depuis le WAN (simulation avec réseau du lycée)

╔═══════════════════════════════════════╗
║ MISSION 2 – RÉPARTITION DE CHARGE WEB ║
╚═══════════════════════════════════════╝
✔ Installation de HAProxy sur SRV-HAPROXY1
✔ Configuration du fichier haproxy.cfg avec balance roundrobin
✔ Les requêtes web sont réparties entre SRV-WEB1 et SRV-WEB2
✔ Redirection de port configurée sur le Stormshield
✔ Test : alternance de réponse entre les deux serveurs lors du rechargement navigateur

╔══════════════════════════════════════╗
║ MISSION 3 – HAUTE DISPONIBILITÉ (HA) ║
╚══════════════════════════════════════╝
✔ Déploiement d’un cluster HeartBeat entre HAPROXY1 et HAPROXY2
✔ Mise en place d’une IP virtuelle (192.168.1.30)
✔ En cas de panne d’un nœud, le second prend le relais automatiquement
✔ Test de bascule effectué en arrêtant manuellement le premier serveur

╔══════════════════════════════════════╗
║ MISSION 4 – VPN SSL POUR TÉLÉTRAVAIL ║
╚══════════════════════════════════════╝
✔ Déploiement d’un serveur VPN OpenVPN
✔ Configuration serveur.conf + certificats TLS
✔ Intégration LDAP avec Active Directory (SRV-ADTECH)
✔ Authentification restreinte au groupe "g_vpn"
✔ Génération de fichiers .ovpn pour les utilisateurs distants

╔═══════════════════════════════════╗
║ MISSION 5 – CONFIGURATION UTM DMZ ║
╚═══════════════════════════════════╝
✔ Configuration des interfaces réseau : WAN, LAN, DMZ
✔ Règles de filtrage Stormshield :
- Accès HTTP/HTTPS vers l’IP virtuelle HAProxy
- Autorisation LDAP/LDAPS vers SRV-ADTECH
- Journalisation des accès activée
✔ NAT et routage configurés pour permettre les échanges LAN ↔ DMZ

╔════════════════════════════════════════╗
║ MISSION 6 – CONTRÔLEUR DE DOMAINE (AD) ║
╚════════════════════════════════════════╝
✔ Installation de SRV-ADTECH (Windows Server 2022)
✔ Domaine créé : techuniverse.lan
✔ Création des OU : Utilisateurs, Groupes, VPN
✔ Comptes nominatifs et groupe "g_vpn" pour l’accès VPN
✔ Résolution DNS et authentification centralisée opérationnelles

╔═════════════════════╗
║ TESTS / VALIDATIONS ║
╚═════════════════════╝
✔ Ping, accès web, et failover HAProxy fonctionnels
✔ Authentification VPN via LDAP : réussite test utilisateur "adurand"
✔ Monitoring manuel effectué sur chaque service

╔══════════════════╗
║ BONNES PRATIQUES ║
╚══════════════════╝
✔ Nommage cohérent des machines
✔ Services segmentés par zones réseau (DMZ, LAN)
✔ Filtrage firewall strict et journalisé
✔ Protocoles d’administration sécurisés (SSH / RDP)

╔═══════════════╗
║ BILAN / RETEX ║
╚═══════════════╝
✔ Infrastructure DMZ hautement disponible et fonctionnelle
✔ Défis techniques : configuration HeartBeat et LDAP
✔ L’ensemble est transposable en production avec peu d’ajustements
Projet complet, bien documenté, couvrant sécurité, HA, télétravail
