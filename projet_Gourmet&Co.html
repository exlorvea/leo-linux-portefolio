PROJET : Modernisation du SI - Gourmet&Co

Réalisé : Avril - Mai 2025
Projet BTS SIO - SISR

╔════════════════════╗
║ CONTEXTE DU PROJET ║
╚════════════════════╝
Gourmet&Co est une entreprise bordelaise de distribution alimentaire haut de gamme pour l'hôtellerie-restauration.
Elle souhaite sécuriser et centraliser son système d’information tout en facilitant la supervision de son infrastructure réseau.

╔══════════════════════════════════════╗
║ MISSION 1 – SEGMENTATION RÉSEAU/VLAN ║
╚══════════════════════════════════════╝
✔ Création de 3 VLAN :
- VLAN 10 (Datacenter) – 192.168.10.0/24
- VLAN 20 (Client LAN) – 192.168.20.0/24
- VLAN 30 (DMZ) – 192.168.30.0/24
✔ Configuration du switch Cisco 2960-X :
- Définition des VLANs, ports d’accès et trunk
✔ Affectation des VLANs sur le Stormshield et activation du routage inter-VLAN
✔ Règles de filtrage pour sécuriser les flux entre zones

╔═════════════════════════════════════╗
║ MISSION 2 – ACTIVE DIRECTORY & DHCP ║
╚═════════════════════════════════════╝
✔ Déploiement de SRV-AD-DHCP sous Windows Server 2022
✔ Promotion en tant que contrôleur de domaine : domaine gourmet.lan
✔ Création d’OU, comptes utilisateurs nominatifs
✔ Configuration DHCP pour le VLAN 20 :
- Étendue : 192.168.20.100 à 192.168.20.200
- DNS : 192.168.20.10 – Passerelle : 192.168.20.1

╔═════════════════════════════════════╗
║ MISSION 3 – DÉPLOIEMENT DE OWNCLOUD ║
╚═════════════════════════════════════╝
✔ Installation d’un serveur OwnCloud sur Debian 12 (SRV-OWNCLOUD)
✔ Configuration Apache2, MariaDB, PHP
✔ Création de la base de données : owncloud / ownclouduser
✔ Configuration VirtualHost Apache, droits www-data appliqués
✔ Intégration LDAP avec l’Active Directory :
- DN : dc=gourmet,dc=lan – Filtre : (objectClass=user)
- Authentification des utilisateurs via AD

╔═══════════════════════════════════════╗
║ MISSION 4 – SUPERVISION AVEC CENTREON ║
╚═══════════════════════════════════════╝
✔ Installation de Centreon sur Debian 12 (SRV-CENTREON)
✔ Ajout des hôtes :
- SRV-AD-DHCP / SRV-OWNCLOUD / UTM-STORMSHIELD
✔ Surveillance des services critiques :
- Ping, HTTP, HTTPS, LDAP, MySQL, Réponse OwnCloud
✔ Interface accessible via : http://192.168.10.10/centreon

╔═════════════════════╗
║ TESTS / VALIDATIONS ║
╚═════════════════════╝
✔ IP via DHCP bien reçue (VLAN 20)
✔ Accès et authentification OwnCloud via AD : OK
✔ Supervision Centreon fonctionnelle et fiable

╔══════════════════╗
║ BONNES PRATIQUES ║
╚══════════════════╝
✔ Nom de machines standardisé
✔ Accès administrateur sécurisé (SSH / RDP)
✔ Utilisation des VLAN pour cloisonner les flux
✔ Comptes nominatifs pour tous les services

╔═══════════════╗
║ BILAN / RETEX ║
╚═══════════════╝
✔ Projet complet validé dans les temps
✔ Apprentissage : LDAP, segmentation réseau, supervision
✘ Problèmes initiaux sur les VLANs (résolus via pare-feu)
✘ Erreur de configuration LDAP corrigée sur OwnCloud
Une infrastructure robuste, documentée, prête à être déployée en production
